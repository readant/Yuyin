# 致谢

余音项目源于大一学年实训课程，最初版本在此阶段完成并提交。

## 项目缘起

大一实训课程要求每组开发一个完整的小型软件项目。作为一个竹笛爱好者，我一直想有一个既能播放音乐、又能辅助练习指法的工具。于是"余音" —— 一个专为竹笛学习者设计的音乐播放与练习平台 —— 就这样诞生了。

从最初的 PyQt6 桌面端原型，到后来重构为 FastAPI Web 应用，再到如今的完整学习工具，这个项目经历了三次大重构、三十余次功能迭代。每一行代码背后，都有老师和队友的汗水与智慧。

## 指导老师

衷心感谢我的两位指导老师：

- **席老师** —— 在项目选题阶段，席老师鼓励我们选择自己真正热爱的方向,具有长远发展的方向。当我说"想做一个竹笛学习工具"时，席老师没有觉得太窄，反而帮我分析了技术可行性和创新点。在架构设计和技术选型方面，席老师一次次帮我们推翻不合理的设计，让项目从一个简单播放器成长为分层的学习平台。
- **张老师** —— 张老师话不多，但总在关键的时候出现。每当我们提交一个阶段的作品，他都会默默看完。每次通知都会给出关键建议，每条都切中要害。实训期间每天签到,偶尔谁迟到了，他不批评，只是在本子上记一下，然后抬头淡淡说一句"下次注意"。正是这种润物无声的监督，让我们整个实训阶段没有一天松懈。

## 学校设施

感谢学校提供的实验室环境、服务器资源和开发工具支持。实训期间，实验室的电脑全天开放，让我们可以随时讨论和调试代码。

## 小组合作

特别感谢与我一同完成这个项目的两位队友：

- **H**（[@Voidhyy](https://github.com/Voidhyy)）—— H 是音频处理的主力。记得有一次深夜，音高检测算法始终不准，H 对比了自相关法、FFT 和 librosa 的 piptrack 三种方案，最终选择了一个在精度和性能之间平衡的实现。那天凌晨三点，他在群里发了一条消息："通了"，后面跟着一段 C4-E4-G4 的识别结果。那种兴奋，至今难忘。

- **G**（[@lnaiad](https://github.com/lnaiad)）—— G 是前端和 UI 设计的主力。从水墨启动动画到洞洞谱的 CSS 布局，G 总能把复杂的交互变成优雅的视觉语言。有一次为了实现"唱片旋转时跟随节拍器节奏变化速度"，G 翻遍了 CSS animation 的文档，最后用 `requestAnimationFrame` 实现了平滑的动态调速。正是 G 的坚持，让余音有了中国古典美学的视觉灵魂。

作为组长，我基本参与了项目的每一个环节 —— 从最初的技术选型和架构设计，到后端 API 开发、数据库设计、Web 前端功能实现，再到后期的测试、文档和部署。因为我入学前就有一定的编程基础，所以承担了技术难度较高的部分：FastAPI 的分层架构、SQLAlchemy ORM 的设计、librosa 音频分析引擎的封装、以及前后端整体的串联和调试。当队友在某个技术点卡住时，我会一起排查问题、给出思路，确保项目不断推进。

三人的分工各有侧重，但更多时候是在交叉协作。代码仓库的每一次 commit、每一个功能模块，都有各自的痕迹。这个项目让我体会到，技术可以一个人钻研，但好产品需要一群人交付。

但更多时候，我们都是边学边做：一起在 StackOverflow 上找答案，一起在实验室通宵调试，一起为一个新功能的实现欢呼。

正是有了你们的共同努力，余音才从一段代码变成了一个真正能帮助竹笛学习者的工具。

## 致谢

这个项目见证了我们大一的成长。从一行行代码到完整的产品，从对 Python 框架一知半解到能够独立设计系统架构，每一步都离不开老师、同学和学校的支持。

感谢所有帮助过我们的人。余音未了，故事继续。
